# Context

This document serves as a glossary and domain map for the DiffToJson project. It contains canonical terms and their definitions, stripped of implementation details.

## Core Domain Terms

### Git Diff
The set of changes introduced by a commit, including added, modified, and deleted lines, and the metadata associated with the affected files.

### JSONL (JSON Lines)
A text format where each single line is a valid JSON object, used here to represent individual commit diffs and their associated metadata for ML training.

### License Detection
The process of identifying the legal license governing a codebase by analyzing `LICENSE*` files, which informs the metadata of the generated JSONL.

### PII Redaction
The identification and removal of Personally Identifiable Information (specifically email addresses) from commit messages to ensure the dataset complies with privacy standards.

### Redaction Tier
The level of PII redaction applied to a training example, controlling which fields (commit message, diff, LLM-generated output) are passed through the redactor before emission.

### ChatML Format
A chat-format schema where a record is a list of messages with `role` (system, user, assistant) and `content` fields. The standard shape used for instruction-tuning and chat fine-tuning datasets.

### Training Example
A record shaped for LLM post-training, comprising a ChatML messages list, provenance metadata, and legal metadata. Distinct from a generic commit record.

### Provenance
The metadata describing where a training example's data came from: the source repository's name and URL. Carried per-record for combination-safety and downstream filtering.

### Legal Metadata
The metadata describing the usage rights of a training example's source code: the license identifier. Carried per-record for the same reasons as provenance.

### Prompt Preset
A named, library-defined template that provides the system and user messages of a training example. The available presets are `default` (generic, style-neutral) and `conventional` (Conventional Commits style).

### Prompt Override
A user-supplied replacement for the system or user message of a training example, provided via CLI flag at extraction time. Overrides take precedence over the selected preset.

### Placeholder
A named variable in a prompt template (e.g., `{diff}`, `{commitMessage}`) substituted with record-specific data at serialization time. Validated strictly: unknown placeholders cause a CLI error.

### LLM Override
A feature where the assistant message of a training example is generated by an LLM at extraction time, rather than taken from the original commit message. Used for data augmentation or style normalization. The original human-written message is preserved in a side field for evaluation.

### Original Assistant Message
The human-written commit message preserved alongside an LLM-generated assistant message when LLM override is enabled. Present only on records produced with override; absent otherwise.
